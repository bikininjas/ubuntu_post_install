# ShellCheck - Vérification Locale

## Installation

```bash
# Ubuntu/Debian
sudo apt install shellcheck

# Ou via snap
sudo snap install shellcheck
```

## Vérification locale avant commit

### Vérifier tous les scripts

```bash
# Depuis la racine du projet
find . -type f -name "*.sh" -not -path "./.git/*" | xargs shellcheck
```

### Vérifier un script spécifique

```bash
shellcheck post_install.sh
shellcheck modules/01-base-system.sh
```

### Vérification détaillée (tous les checks)

```bash
shellcheck -x -o all post_install.sh
```

### Vérifier avec la configuration du projet

La configuration ShellCheck est dans `.shellcheckrc` et sera automatiquement utilisée.

## CI/CD

Les workflows GitHub Actions vérifient automatiquement :
- `.github/workflows/shellcheck.yml` - Vérification ShellCheck
- `.github/workflows/ci.yml` - Validation complète (syntaxe, permissions, structure)

## Configuration

Le fichier `.shellcheckrc` désactive certains avertissements :
- `SC2162` - read sans -r (pour saisie de mots de passe)
- `SC1090/SC1091` - Fichiers sourcés dynamiquement
- `SC2312` - Pipelines (style idiomatique Bash)
- Et autres...

## Commandes rapides

```bash
# Vérifier tout avant de commit
./check-shellcheck.sh

# Voir les erreurs avec numéros de ligne
shellcheck -f gcc modules/*.sh

# Ignorer un avertissement spécifique dans un script
# shellcheck disable=SC2312
command | pipeline
```
